Linux下搭建Hexo个人博客

发表于 2020-12-01 更新于 2020-12-04 分类于 Hexo

本博基于 ubuntu18.04+git_v2.17.1+NodeJS_v8.10.0+hexo_v3.9.0 不同平台和版本实际操作可能略不同。

准备工作

安装依赖

安装Git

安装npm

安装Node.js

安装Hexo

执行init命令初始化hexo,生成blog文件，就是你的博客根目录，所有的操作都在里面进行。

生成静态网页

启动本地服务，进行文章预览调试

浏览器输入 http://localhost:4000

配置GitHub

在GitHub页面建立Repository

GitHub页面建立与你用户名对应的仓库，仓库名必须为 yourname.github.io ，固定写法

建立关联 bash vi /users/blog/_config.yml

翻到最下面，改成：

执行命令： bashnpm install --save hexo-deployer-git

然后，执行配置命令：hexo deploy

在浏览器中输入 http://yourname.github.io/

每次部署命令:

配置自己的域名

首先设置域名DNS解析，Type 改成 CNAME, Target 填上 yourname.github.io, 保存

在 source 下创建一个名为 CNAME 的文件，内容为你要使用的域名，如本博客为 iinine.github.io

编辑 _config.yml，使 url 的内容为你的主页地址，如本博客为 https://iinine.github.io

部署到 git 后，进入 github，打开网站所在项目，点击 Settings/GitHub Pages ,勾选 Enforce HTTPS

常用命令

简化：hexo n 新建文章

简化：hexo n p新建页面

清理缓存

简化：hexo g 生成静态页面

简化：hexo s 开启预览访问端口

简化：hexo d 将.deploy目录部署到GitHub

博客文章在 /blog/source/_posts 文章支持Markdown语法。如果想修改头像可以直接在主题的 _config.yml 文件里面修改，友情链接，之类的都在这里。

上面的是Linux搭建方法，相信大家用Windows的比较多。推荐朋友的Windows搭建教程
